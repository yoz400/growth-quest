## 1. 背景

召喚オンボーディングの「冒険者を選ぶ」画面（2026-07-03 実機評価で確認）:

1. **名前が無機質**:「冒険者A/B/C」。妖精・召喚の儀という世界観と不釣り合い
2. **選択状態が曖昧**: AとBの両方に枠が付いて見え、どれが選ばれているか分からない
3. アバター画面の切替ボタン（app.js 8681付近 `av-type-btn`）も「冒険者A 男性風・短髪」表記

## 2. 設計

### 2.1 名前と肩書き（データは互換維持）

保存値は今まで通り `'A' | 'B' | 'C'`。**表示名だけ**マッピングで変える:

```javascript
const ADVENTURERS = {
  A: { name: 'レン', title: '凜々しい剣士',     desc: 'まっすぐ突き進む努力家' },
  B: { name: 'ミア', title: '聡明な魔法使い',   desc: 'コツコツ積み上げる知性派' },
  C: { name: 'ソラ', title: '旅する吟遊詩人',   desc: '自由に楽しく続ける自由人' },
};
```

- 名前はヨージが変更してよい（実装前に確認。決まらなければ上記案で進める）
- 適用箇所: ①召喚の選択カード（app.js 10380付近 `types` 配列）
  ②アバター画面の `av-type-btn`（8681付近） ③その他「冒険者A」表記を grep して全置換

### 2.2 選択状態の明確化（召喚カード）

```text
未選択:  枠なし・彩度そのまま・少し暗く(opacity .75)
選択中:  シアンの枠＋外側グロー＋右上に ✓ バッジ＋名前がシアンに
```

- 選択は常に**ちょうど1人**。初期状態から A を選択済みにする（現状踏襲）
- カードに `aria-pressed="true/false"` を付与（選択UIの読み上げ対応）
- CSS例: `.summon-char.selected { border:2px solid var(--cyan); box-shadow:0 0 18px rgba(103,232,249,.35); }`
  `.summon-char:not(.selected) { opacity:.75; }` ＋ `.summon-char-check`（✓バッジ）

### 2.3 選んだ名前を体験に反映（小さく世界観を強める）

- 召喚完了画面のセリフに名前を入れる:「ようこそ、剣士レン。」
- 冒険者名（ユーザーが入力する自分の名前）とは別物なので混同しない表記にする
  （例: カード下部に「相棒: レン」）

## 3. 受け入れ基準

- [ ] 「冒険者A/B/C」表記が画面上から消え、名前＋肩書きになっている
- [ ] 選択中のカードが1枚だけ明確に光り、未選択は暗い
- [ ] 既存ユーザーのデータ（avatarType）がそのまま引き継がれる
- [ ] アバター画面の切替ボタンも新名称になっている
- [ ] カードに aria-pressed が付いている

## 4. テスト手順

1. localStorage クリア → 召喚でカードを順にタップ → 常に1枚だけ選択表示
2. 選択して完了 → ホームのアバターが選んだ冒険者になっている
3. 既存データ(avatarType:'B')で開く → ミアが表示される
4. アバター画面で切替 → 名前表記が正しい
