## 1. まずヨージに確認すること（勝手に消さない）

`assets/otomon/roster/` に **「pet_020 2.png」のような "2" 付き複製が21個**（未コミット）と、
**変更済みだが未コミットのPNGが30個**ある。これはヨージの画像作業の途中経過の可能性が高い。

- 質問①: 「pet_XXX 2.png」は消してよいか？（本体に反映済みの複製なら削除）
- 質問②: 変更済みPNG30個はこの内容でコミットしてよいか？
- **回答をもらうまで §2 以降に進まない。**

## 2. 現状と目標

| 項目 | 現状 | 目標 |
|------|------|------|
| assets 合計 | 166MB | 50MB以下 |
| 形式 | PNG中心 | WebP（品質85）＋PNGフォールバック不要（全モダンブラウザ対応済み） |

> **WebP** とは：PNGより5〜10倍小さくなる画像形式。見た目の劣化はほぼ分からない。

## 3. 作業手順（スクリプト化する）

1. 重複削除: `find assets -name "* 2.png"` で一覧→ヨージ確認済みなら削除
2. 変換スクリプト `tools/convert_webp.sh` を作成:
   - `cwebp -q 85` で assets/ 配下の PNG を一括変換（元PNGはまず残す）
   - 変換前後のサイズ比較レポートを出力
3. 参照の書き換え: `.png` を参照している箇所を grep（index.html / app.js / otomon.js /
   manifest.json / CSS）して `.webp` に置換。
   ⚠️ 例外: **PWAアイコン（assets/icons/）とapple-touch-iconはPNGのまま**（iOS要件）
4. 動作確認後、元PNGを削除する**別コミット**を作る（戻しやすくするため）
5. `cwebp` が無ければ `brew install webp`

## 4. 受け入れ基準

- [ ] ヨージの確認①②が取れている

- [ ] PWAアイコンはPNGのまま
